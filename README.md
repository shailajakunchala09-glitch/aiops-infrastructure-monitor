# Screenshots

These five screenshots are included in the repo (`01-overview.png` through
`05-servers.png`) and are static mockups of the dashboard UI, styled with
the project's actual `tokens.css` / `components.css`, and populated with
representative demo data.

They're a placeholder for real screenshots: once you run
`docker compose up`, register a few servers, and let the monitoring agent
report real metrics for a bit, replace these with live captures of your own
running instance (same filenames, so the root `README.md` keeps working).
Recommended: 1440x900+, PNG, browser chrome cropped out. You can also add
`06-swagger.png` — a capture of `/api/docs` — to show the auto-generated API
documentation.
