# API Documentation

Interactive, always-current API documentation is auto-generated by FastAPI
from the route definitions and Pydantic schemas:

- **Swagger UI:** `http://localhost:8000/api/docs`
- **ReDoc:** `http://localhost:8000/api/redoc`
- **Raw OpenAPI schema:** `http://localhost:8000/api/openapi.json`

This document summarizes the endpoint surface for quick reference. All
endpoints are prefixed with `/api/v1`.

## Authentication

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| POST | `/auth/register` | none | Create a new platform user. |
| POST | `/auth/login` | none | Exchange credentials for an access + refresh JWT. |

All subsequent user-facing endpoints require `Authorization: Bearer <access_token>`.
Agent ingest endpoints instead require an `X-API-Key` header issued at server
registration.

## Infrastructure Monitoring (Servers)

| Method | Endpoint | Roles | Description |
|---|---|---|---|
| POST | `/servers` | ADMIN, SRE | Register a new monitored server; returns a one-time agent API key. |
| GET | `/servers` | any authenticated | List registered servers, optional `environment` filter. |
| GET | `/servers/{id}` | any authenticated | Get a single server's detail. |
| PATCH | `/servers/{id}` | ADMIN, SRE | Update hostname/thresholds. |
| DELETE | `/servers/{id}` | ADMIN | Deregister a server. |

## Metrics

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| POST | `/metrics/ingest` | Agent API key | Submit a metrics sample; triggers threshold evaluation, alerting, and status update. |
| GET | `/metrics/servers/{id}` | JWT | Time-series metrics for a server over `?hours=` window. |
| GET | `/metrics/servers/{id}/summary` | JWT | Aggregated avg/max CPU, memory, disk for a window. |

## Log Management

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| POST | `/logs/ingest` | Agent API key | Submit a log line; auto-creates an incident after repeated CRITICAL entries. |
| GET | `/logs` | JWT | Search logs by `server_id`, `level`, `keyword`, `hours`. |
| GET | `/logs/analytics/level-distribution` | JWT | Count of logs per level in a window (pie chart data). |
| GET | `/logs/analytics/top-errors` | JWT | Applications generating the most ERROR/CRITICAL logs (failure analysis). |

## Incident Management

| Method | Endpoint | Roles | Description |
|---|---|---|---|
| POST | `/incidents` | any authenticated | Manually open an incident. |
| GET | `/incidents` | any authenticated | List incidents; filter by `status`, `severity`, `server_id`. |
| GET | `/incidents/stats` | any authenticated | Open/resolved/critical counts + average resolution time (KPI cards). |
| GET | `/incidents/trends/monthly` | any authenticated | Monthly incident counts for trend charts. |
| GET | `/incidents/{id}` | any authenticated | Get incident detail. |
| PATCH | `/incidents/{id}` | ADMIN, SRE, OPERATOR | Update status/assignment/resolution notes; timestamps acknowledgement/resolution automatically. |

## Error Format

Validation errors (HTTP 422) return:

```json
{
  "detail": "Input validation failed",
  "errors": [ { "loc": ["body", "cpu_percent"], "msg": "...", "type": "..." } ]
}
```

Unhandled server errors (HTTP 500) return a generic message and are logged
server-side with full context rather than leaking internals to the client.

## Example: Registering a Server and Sending a Metric

```bash
# 1. Log in as an admin
TOKEN=$(curl -s -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@aiops.local","password":"Admin@12345"}' | jq -r .access_token)

# 2. Register a server
curl -s -X POST http://localhost:8000/api/v1/servers \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"hostname":"prod-web-05","ip_address":"10.0.1.15","environment":"PRODUCTION"}'
# -> returns { ..., "api_key": "..." }

# 3. Send a metric sample as that server's agent
curl -s -X POST http://localhost:8000/api/v1/metrics/ingest \
  -H "X-API-Key: <api_key from step 2>" -H "Content-Type: application/json" \
  -d '{"cpu_percent": 92.5, "memory_percent": 71.0, "disk_percent": 55.0}'
```
